Diagrama Detallado de Casos de Uso

Los casos de uso del sistema Crescendo definen las interacciones entre los actores y el sistema para lograr objetivos específicos.

Actores del Sistema

Administrador del Sistema

Responsable de la configuración, monitoreo y mantenimiento del sistema.

  • Configurar parámetros del sistema
  • Monitorear procesos
  • Generar reportes
  • Gestionar usuarios

Sistema Automatizado

Procesos automáticos que ejecutan las tareas principales del sistema.

  • Procesar archivos CSV
  • Validar traspasos
  • Generar colas de prioridad
  • Crear respaldos

Entidades Gubernamentales

Sistemas externos que proporcionan información de embargos.

  • Enviar archivos de embargos
  • Actualizar información
  • Recibir notificaciones

Fondo Crescendo

Sistema principal que recibe los resultados del procesamiento.

  • Proporcionar datos de cotizantes
  • Recibir traspasos aprobados
  • Consultar estados

Casos de Uso Principales

UC-001: Procesar Traspaso de Cotizante

Alta Prioridad
Descripción:

El sistema procesa automáticamente las solicitudes de traspaso de cotizantes desde fondos privados hacia Crescendo.

Actor Principal:

Sistema Automatizado

Precondiciones:
  • Archivo cotizantes.csv disponible
  • Sistema iniciado correctamente
  • Conexión a base de datos activa
Flujo Principal:
  1. Sistema lee archivo cotizantes.csv
  2. Valida formato y datos de cada cotizante
  3. Aplica reglas de elegibilidad
  4. Calcula prioridad para cotizantes válidos
  5. Agrega a cola de procesamiento
  6. Genera archivo de resultados
  7. Registra transacción en auditoría
Postcondiciones:
  • Traspasos procesados y priorizados
  • Archivo de resultados generado
  • Log de auditoría actualizado

UC-002: Gestionar Lista Negra de Embargos

Alta Prioridad
Descripción:

El sistema mantiene y actualiza una lista negra de cotizantes con embargos activos.

Actor Principal:

Sistema Automatizado

Precondiciones:
  • Archivo embargos.csv disponible
  • Lista enlazada inicializada
  • Conexión SFTP establecida
Flujo Principal:
  1. Sistema descarga archivo embargos.csv
  2. Procesa cada registro de embargo
  3. Actualiza lista enlazada de embargos
  4. Verifica cotizantes contra lista negra
  5. Genera archivo lista_negra.csv
  6. Cifra archivo de salida
Postcondiciones:
  • Lista negra actualizada
  • Archivo cifrado generado
  • Cotizantes embargados identificados

UC-003: Sincronizar con Entidades Gubernamentales

Media Prioridad
Descripción:

El sistema se conecta diariamente con entidades gubernamentales para obtener información actualizada de embargos.

Actor Principal:

Sistema Automatizado

Actor Secundario:

Entidades Gubernamentales

Flujo Principal:
  1. Sistema inicia conexión SFTP programada
  2. Autentica con credenciales seguras
  3. Descarga archivos actualizados
  4. Valida integridad de archivos
  5. Procesa nuevos embargos
  6. Actualiza base de datos local
  7. Confirma recepción exitosa

UC-004: Generar Reportes de Auditoría

Media Prioridad
Descripción:

El administrador genera reportes detallados de las operaciones del sistema para auditoría y control.

Actor Principal:

Administrador del Sistema

Flujo Principal:
  1. Administrador accede al panel de control
  2. Selecciona tipo de reporte
  3. Define rango de fechas
  4. Sistema consulta logs de auditoría
  5. Genera reporte en formato solicitado
  6. Presenta resultados al administrador

Principios de Diseño de Interfaz
