Diseño de Bases de Datos

Diseño lógico, modelo entidad-relación, scripts y sistema de auditoría de la base de datos

Diseño Lógico

El diseño lógico de la base de datos define la estructura conceptual de los datos, las relaciones entre entidades y las reglas de negocio que rigen el sistema.

Principios del Diseño

Normalización

Base de datos normalizada hasta la tercera forma normal (3NF) para eliminar redundancias y garantizar integridad de datos.

  • Primera Forma Normal: Eliminación de grupos repetitivos
  • Segunda Forma Normal: Eliminación de dependencias parciales
  • Tercera Forma Normal: Eliminación de dependencias transitivas

Integridad Referencial

Implementación de restricciones de integridad para mantener la consistencia de los datos.

  • Claves primarias únicas en todas las tablas
  • Claves foráneas con restricciones CASCADE
  • Restricciones CHECK para validación de datos

Optimización

Diseño optimizado para consultas frecuentes y operaciones de alto volumen.

  • Índices estratégicos en columnas de búsqueda
  • Particionamiento de tablas grandes
  • Desnormalización controlada para reportes

Escalabilidad

Estructura preparada para crecimiento futuro y aumento de volumen de datos.

  • Diseño modular y extensible
  • Soporte para replicación
  • Estrategias de archivado de datos históricos

Scripts de Base de Datos

Scripts SQL para la creación, configuración y mantenimiento de la base de datos del sistema Crescendo.

Script de Creación de Base de Datos

-- Creación de la base de datos
CREATE DATABASE crescendo_db
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;

USE crescendo_db;

-- Configuración de zona horaria
SET time_zone = '-05:00'; -- Colombia

-- Configuración de modo SQL
SET sql_mode = 'STRICT_TRANS_TABLES,NO_ZERO_DATE,NO_ZERO_IN_DATE,ERROR_FOR_DIVISION_BY_ZERO';

Creación de Tablas Principales

-- Tabla de Entidades Embargantes
CREATE TABLE entidades_embargantes (
    id_entidad INT AUTO_INCREMENT PRIMARY KEY,
    nombre_entidad VARCHAR(100) NOT NULL,
    tipo_entidad ENUM('JUDICIAL', 'FISCAL', 'ADMINISTRATIVO', 'COOPERATIVO') NOT NULL,
    nit VARCHAR(15) UNIQUE NOT NULL,
    direccion TEXT,
    telefono VARCHAR(15),
    email_contacto VARCHAR(100),
    activa BOOLEAN DEFAULT TRUE,
    fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_tipo_entidad (tipo_entidad),
    INDEX idx_activa (activa)
);

-- Tabla de Cotizantes
CREATE TABLE cotizantes (
    id_cotizante VARCHAR(20) PRIMARY KEY,
    numero_documento VARCHAR(15) UNIQUE NOT NULL,
    tipo_documento ENUM('CC', 'CE', 'TI', 'PP', 'NIT') NOT NULL DEFAULT 'CC',
    nombre_completo VARCHAR(100) NOT NULL,
    fecha_nacimiento DATE NOT NULL,
    edad INT GENERATED ALWAYS AS (YEAR(CURDATE()) - YEAR(fecha_nacimiento)) STORED,
    genero ENUM('M', 'F', 'O') NOT NULL,
    telefono VARCHAR(15),
    email VARCHAR(100),
    direccion TEXT,
    ciudad VARCHAR(50),
    tiempo_fondo_privado INT DEFAULT 0,
    historial_deudas BOOLEAN DEFAULT FALSE,
    estado_traspaso ENUM('PENDIENTE', 'EN_PROCESO', 'COMPLETADO', 'RECHAZADO') DEFAULT 'PENDIENTE',
    fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    fecha_actualizacion TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_documento (numero_documento),
    INDEX idx_estado (estado_traspaso),
    INDEX idx_ciudad (ciudad)
);

Auditoría

El sistema de auditoría de Crescendo proporciona trazabilidad completa y cumplimiento normativo para todas las operaciones del sistema.

Componentes del Sistema de Auditoría

Trazabilidad Completa

Registro detallado de todas las operaciones con información contextual completa y timestamps precisos.

Integridad de Datos

Protección criptográfica de logs para prevenir manipulación y garantizar autenticidad.

Cumplimiento Normativo

Alineación con regulaciones de la Superintendencia Financiera y estándares internacionales.

Alertas Automáticas

Detección proactiva de actividades sospechosas y operaciones críticas en tiempo real.

Estructura de Auditoría

Tabla de Auditoría Detallada

CREATE TABLE logs_auditoria_detallada (
    id_log BIGINT AUTO_INCREMENT PRIMARY KEY,
    timestamp TIMESTAMP(6) DEFAULT CURRENT_TIMESTAMP(6),
    id_usuario VARCHAR(50),
    ip_origen VARCHAR(45),
    
    -- Contexto de la operación
    modulo VARCHAR(50) NOT NULL,
    operacion VARCHAR(100) NOT NULL,
    tipo_evento ENUM('CREATE', 'READ', 'UPDATE', 'DELETE', 'LOGIN', 'LOGOUT') NOT NULL,
    
    -- Entidades afectadas
    id_cotizante VARCHAR(20),
    id_traspaso VARCHAR(20),
    tabla_afectada VARCHAR(50),
    
    -- Detalles de la operación
    datos_anteriores JSON,
    datos_nuevos JSON,
    
    -- Resultado y métricas
    resultado ENUM('EXITOSO', 'FALLIDO', 'PARCIAL') NOT NULL,
    duracion_ms INT,
    nivel_criticidad ENUM('BAJA', 'MEDIA', 'ALTA', 'CRITICA') DEFAULT 'MEDIA',
    
    INDEX idx_timestamp (timestamp),
    INDEX idx_usuario (id_usuario),
    INDEX idx_operacion (operacion),
    INDEX idx_criticidad (nivel_criticidad)
);

Gestión de Cotizantes

  • Cambios de prioridad en cola
  • Aprobación/Rechazo con justificación

Gestión de Embargos
  • Registro y actualización de embargos
  • Levantamiento con documentación
  • Consultas por entidades autorizadas
  • Reportes de cumplimiento
Seguridad del Sistema
  • Autenticación exitosa y fallida
  • Cambios de permisos y roles
  • Accesos no autorizados
  • Configuración del sistema

Monitoreo y Alertas

Alertas Automáticas
  • Actividad Sospechosa: Múltiples intentos fallidos, accesos fuera de horario
  • Operaciones Críticas: Traspasos por montos altos, cambios en embargos
  • Cumplimiento: Traspasos pendientes, embargos por vencer
Métricas de Rendimiento
  • Tiempo de Respuesta: Latencia en operaciones críticas
  • Volumen: Carga del sistema y picos de actividad
  • Disponibilidad: Uptime y tiempo de recuperación

  • Estándares Internacionales: ISO 27001, SOX, COBIT 5

Configuración y Mantenimiento

Retención de Datos
  • Logs Operativos: 2 años activos, 7 años archivados
  • Logs de Seguridad: 5 años activos, archivo permanente para eventos críticos
Seguridad y Protección
  • Integridad: Hash SHA-256 y firma digital para logs críticos
  • Inmutabilidad: Registros no modificables una vez creados
  • Control de Acceso: Principio de menor privilegio y autenticación multifactor
